1. 
Это механизм в js который позволяет событиям двигаться по дереву DOM: от верхнего уровня до цели и обратно.

Существует три фазы:
Capture phase — когда событие движется в сторону целевого элемента.
Target phase — когда событие достигло целевого элемента
Bubling phase — когда событие движется от целевого элемента.

Способы применения знаний о фазах распространения событий:
- Использование фазы захвата для предотвращения выполнения других обработчиков, если это необходимо.
- Создание сложных интерфейсов, где события могут быть перехвачены на разных уровнях иерархии элементов.
- Оптимизация производительности путем делегирования событий на родительские элементы вместо назначения их на каждый дочерний элемент.

2.
Promise — это объект представляющий результат асинхронной операции: либо успешный результат, либо ошибку. Проще говоря, это обещание, что код когда-нибудь выполнится и выдаст результат. Промисы нужны для работы с асинхронными задачами — например, загрузкой данных с сервера, чтением файлов или чем-то ещё, что занимает время. Пока промис в ожидании, остальной код спокойно выполняется дальше и не зависает. 

Промис может находиться в одном из трёх состояний

Pending — ожидание (промис ещё не завершился).
Fulfilled — успешно завершён.
Rejected — ошибка.
 
Также асинхронный код можно обрабатывать с помощью колбэков и async/await.

Event Loop следит за стеком вызовов и очередью сообщений. Когда стек вызовов пуст, он берет первое сообщение из очереди и выполняет его, что позволяет обрабатывать асинхронные операции и гарантирует, что код выполняется последовательно.

 
3.
ООП (Объектно-ориентированное программирование) парадигма программирования позволяющая моделировать реальный мир в коде. Представляя сущности как «объекты» с характеристиками (данными) и поведением (функциями). 

Ключевые принципы ООП:

 Инкапсуляция - позволяет скрыть детали реализации объекта и предоставить доступ к данным только через определённые публичные методы. В js реализовано с помощью замыканий, функций, классов, приватных полей.

 Наследование - позволяет создавать новые классы на основе существующих, и они будут наследовать его свойства и методы. В js реализованно с помощью прототипов и ключевого слова extends в классах

 Полиморфизм - позволяет использовать один интерфейс для работы с разными типами объектов. В js реализуется за счёт переопределения методов. Подклассы могут переопределять методы, унаследованные от своих родительских классов.



4. Обработка URL браузером

Процесс обработки URL браузером включает несколько этапов:

1. Пользователь вводит URL в адресную строку браузера.

2. Браузер отправляет запрос на DNS-сервер для получения IP-адреса сервера.

4. Браузер устанавливает TCP-соединение с сервером по полученному IP-адресу.

5. Браузер отправляет HTTP-запрос на сервер для запрашиваемого ресурса.

6. Сервер обрабатывает запрос и отправляет ответ обратно в браузер.

7. Браузер получает HTTP-ответ от сервера.

8. Браузер анализирует HTML-код, загружает необходимые ресурсы (CSS, JS, изображения) и отображает страницу пользователю.
